# CORTEX Documents Organization

This directory contains all informational documents generated by CORTEX and Copilot interactions, organized in a structured hierarchy.

## üìÅ Directory Structure

### `/reports/`
**Purpose:** Formal status reports and completion documents
- Implementation reports (e.g., `CORTEX-3.0-FINAL-IMPLEMENTATION-REPORT.md`)
- Phase completion reports (e.g., `PHASE-0-COMPLETION-REPORT.md`)
- Track completion documents (e.g., `CORTEX-2.1-TRACK-A-COMPLETE.md`)
- Release reports and status updates

**Naming Convention:** `[COMPONENT]-[VERSION]-[TYPE]-REPORT.md`
- Example: `CORTEX-3.0-IMPLEMENTATION-REPORT.md`

### `/analysis/`
**Purpose:** Deep-dive analysis documents and investigations
- Investigation reports (e.g., `INVESTIGATION-ANALYSIS-REPORT.md`)
- Architecture analysis
- Performance analysis
- Problem analysis documents

**Naming Convention:** `[TOPIC]-ANALYSIS-[DATE].md`
- Example: `ROUTER-PERFORMANCE-ANALYSIS-2025-11-14.md`

### `/summaries/`
**Purpose:** Quick overview and summary documents
- Implementation summaries
- Feature summaries
- Meeting summaries
- Daily/weekly progress summaries

**Naming Convention:** `[COMPONENT]-SUMMARY-[DATE].md`
- Example: `TIER3-IMPLEMENTATION-SUMMARY-2025-11-14.md`

### `/investigations/`
**Purpose:** Investigation documents and research findings
- Feature investigations
- Architecture investigations  
- Technology research
- Problem investigation documents

**Naming Convention:** `[TOPIC]-INVESTIGATION-[STATUS].md`
- Example: `CORTEX-3.0-INVESTIGATION-ARCHITECTURE-COMPLETE.md`

### `/planning/`
**Purpose:** Planning documents and roadmaps
- Implementation plans
- Feature planning documents
- Architecture planning
- Roadmap documents

**Naming Convention:** `[COMPONENT]-[TYPE]-PLAN.md`
- Example: `CORTEX-3.0-ARCHITECTURE-PLAN.md`

### `/conversation-captures/`
**Purpose:** Strategic conversation captures for learning
- High-value conversation transcripts
- Design decision conversations
- Problem-solving sessions
- Learning-rich interactions

**Naming Convention:** `CONVERSATION-CAPTURE-[DATE]-[TOPIC].md`
- Example: `CONVERSATION-CAPTURE-2025-11-14-AUTHENTICATION.md`

### `/implementation-guides/`
**Purpose:** How-to guides and implementation instructions
- Setup guides
- Integration guides
- Feature implementation guides
- Quick reference guides

**Naming Convention:** `[TOPIC]-[TYPE]-GUIDE.md`
- Example: `CORTEX-INTEGRATION-GUIDE.md`
