## PURPOSE AND SCOPE

The HealthEquity, Inc. (“Company”) *Information Technolog**y** Policy* (“Policy”) sets forth the policies regarding information technology operations, services, and resources in the Company’s environment. This Policy applies to all Company systems, applications, or platforms used for storing, processing or transmitting information, including network and computer hardware, software and applications, mobile devices, across all Company working environments.


The goals of this Policy include the following:


Define the roles, responsibilities, and critical elements for the efficient operations and support of the Company’s IT systems.

Ensure the secure and efficient use of IT resources.

Protect sensitive and confidential information.

Comply with legal and regulatory requirements.

Define the responsibilities of users and IT staff.

## Definitions

When used in the context of this Policy, the following definitions apply.



| Term | Definition |
| --- | --- |
| Asset | Something with monetary value. |
| Change | An alteration to our environment that has risk. |
| Configuration Item (CI) | Configuration item, a single item in a CMDB.
For example, a server, a piece of software, or a laptop. |
| CMDB | Configuration Management Database.
A centralized repository used to store information about the components of an IT system, known as configuration items (CIs). 
These items can include hardware, software, network devices, and documentation. |
| Disaster | An incident usually involving multiple services. |
| Impact | The measured, categorized severity and scope and incident has on business processes, services, or users. |
| Incident | The unplanned loss or reduction in quality of an IT service. |
| Major Incident | A high-impact, urgent issue that significantly disrupts business operations or affects a large number of users. 
The issue(s) involve critical services becoming unavailable, leading to substantial business impact, such as loss of revenue, reputation damage, or operational downtime. |
| Outage | The planned or unplanned loss of a service. |
| Priority | A value determined by impact and urgency. |
| Problem | The underlying cause of one or more incidents. |
| Risk | A combination of a negative event likelihood and its scope of impact are evaluated together to create a risk profile. |
| Root Cause Analysis (RCA) | A step in the problem process to identify underlying issues responsible foran incident, and develop measures to prevent recurrence. |
| Service | A value add to the company, a combination of people, process and technology. |
| Urgency | The speed or velocity at which an incident needs to be resolved to minimized the negative effects on business operations. |

## REQUIREMENTS

Information Technology (IT) provides computing services that are consumed by external partners, clients, and members, and internal groups, member/client services, People, Legal, etc. This Policy and related Technology Standards govern the Company’s IT services and help ensure a stable, predictable, secure, and cost-effective environment.  

### Acceptable Use

General Use: IT resources are provided for business purposes. Personal use should be limited and must not interfere with work responsibilities.

Prohibited Activities: Users must not engage in activities that are illegal, unethical, or that could harm the company's reputation or IT infrastructure.

### Data Security

Data Classification: All data must be classified according to its sensitivity (e.g., public, internal, confidential).

Data Protection: Confidential data must be encrypted, and access must be restricted to authorized personnel only.

Data Backup: Regular backups must be performed to ensure data integrity and availability.

### Network Security

Access Control: Network access must be controlled through authentication mechanisms such as passwords and multi-factor authentication.

Monitoring: Network activity must be monitored to detect and respond to security incidents.

### Software Management

Software Installation: Only authorized software may be installed on company devices. Unauthorized software is prohibited.

Software Updates: All software must be kept up-to-date with the latest security patches and updates.

### Hardware Management

Asset Management: All IT hardware must be inventoried and tracked.

Maintenance: Regular maintenance must be performed to ensure hardware reliability and performance.

### Incident Response

Reporting: All security incidents must be reported immediately to the IT department.

Response Plan: The IT department must have a documented incident response plan to address and mitigate security incidents.
