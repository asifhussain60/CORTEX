    CORTEX Folder Structure Design¶

    Version: 1.0
    Date: 2025-11-06
    Status: üéØ DESIGN APPROVED - Ready to Implement

    Design Principles¶

    1. No Root Clutter¶

    • Every file has a designated folder home
    • Root contains ONLY: README.md, LICENSE, .gitignore, core config files
    • No orphaned scripts, docs, or data files in root

    2. Clear Hierarchy¶

    • Maximum 3 levels deep (except node_modules, .git)
    • Self-documenting folder names
    • Consistent naming conventions

    3. Separation of Concerns¶

    • Source code separate from documentation
    • Tests mirror source structure
    • Configuration centralized
    • Data/artifacts isolated

    File Placement Rules¶

    When creating ANY new file, ask:¶

    1. Is it source code? ‚Üí brain/ or dashboard/src/
    2. Is it a test? ‚Üí tests/ (mirror source structure)
    3. Is it configuration? ‚Üí config/
    4. Is it documentation? ‚Üí docs/
    5. Is it a script? ‚Üí scripts/
    6. Is it a prompt? ‚Üí prompts/
    7. Is it data/artifacts? ‚Üí Appropriate data folder or .gitignore

    NEVER:¶

    • ‚ùå Create files in project root (except core configs)
    • ‚ùå Create ad-hoc folders without design
    • ‚ùå Mix concerns (code + docs in same folder)
    • ‚ùå Nest more than 3 levels (except node_modules, .git)

    Enforcement¶

    Pre-commit Hook Rule¶

    Add to .git/hooks/pre-commit:

    # Check for root clutter (files not in allowed list)
ALLOWED_ROOT="README.md LICENSE .gitignore cortex.config.json package.json"

for file in *; do
  if [[ -f "$file" ]] && ! echo "$ALLOWED_ROOT" | grep -q "$file"; then
    echo "‚ùå ERROR: File '$file' not allowed in root!"
    echo "   Move to appropriate folder (see DIRECTORY-STRUCTURE.md)"
    exit 1
  fi
done
